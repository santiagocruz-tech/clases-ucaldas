1. Media Queries @media

Reglas CSS que se activan según el ancho de pantalla. Este ejemplo usa Mobile First: el CSS base es para móvil y los @media (min-width) agregan estilos para pantallas más grandes.

📱 Móvil < 576px
📋 Tablet 576px - 768px
💻 Escritorio 768px - 1024px
🖥️ Pantalla grande > 1024px 
/* ===== MOBILE FIRST ===== */

/* Base: estilos para MÓVIL (sin media query) */
.elemento {
  font-size: 14px;
  padding: 12px;
}

/* Tablet: 576px en adelante */
@media (min-width: 576px) {
  .elemento {
    font-size: 16px;
    padding: 16px;
  }
}

/* Escritorio: 768px en adelante */
@media (min-width: 768px) {
  .elemento {
    font-size: 18px;
    padding: 24px;
  }
}

/* Pantalla grande: 1024px en adelante */
@media (min-width: 1024px) {
  .elemento {
    font-size: 20px;
    padding: 32px;
  }
}

/* Breakpoints comunes:
   576px  → Móvil grande / Tablet pequeña
   768px  → Tablet
   1024px → Escritorio
   1200px → Escritorio grande
*/

2. Unidades relativas

Las unidades fijas (px) no se adaptan. Las relativas (%, em, rem, vw, vh) sí lo hacen.

%

Relativo al elemento padre

75%

rem

Relativo al font-size del <html> (16px por defecto)

1rem = 16px 1.5rem = 24px

em

Relativo al font-size del elemento padre

Padre: 18px
Hijo: 1.5em = 27px

vw / vh

Relativo al viewport (1vw = 1% del ancho de ventana)

50vw de ancho
/* Unidades relativas más usadas */

/* % → relativo al padre */
.hijo { width: 75%; }

/* rem → relativo al html (consistente en toda la página) */
.titulo { font-size: 2rem; }     /* 2 × 16px = 32px */
.texto  { font-size: 1rem; }     /* 1 × 16px = 16px */
.small  { font-size: 0.875rem; } /* 0.875 × 16px = 14px */

/* em → relativo al padre (puede acumularse, cuidado) */
.padre { font-size: 18px; }
.hijo  { font-size: 1.5em; }  /* 1.5 × 18px = 27px */

/* vw/vh → relativo al viewport */
.hero { height: 50vh; }   /* 50% de la altura de la ventana */
.ancho { width: 80vw; }   /* 80% del ancho de la ventana */

/* ✅ Recomendación:
   - rem para font-size (predecible)
   - % para anchos de contenedores
   - vw/vh para secciones hero o full-screen
*/

3. Tipografía responsiva con clamp()

clamp(mínimo, preferido, máximo) crea texto que escala fluidamente con el viewport, sin saltos bruscos.

Título principal

Subtítulo de sección

Este párrafo usa clamp() para su tamaño de fuente. Redimensioná la ventana y observá cómo el texto crece y decrece fluidamente entre el mínimo y el máximo. 

/* clamp(mínimo, preferido, máximo) */

/* Título: mínimo 1.5rem, crece con el viewport, máximo 3rem */
h1 {
  font-size: clamp(1.5rem, 4vw + 0.5rem, 3rem);
}

/* Subtítulo */
h2 {
  font-size: clamp(1.2rem, 2.5vw + 0.5rem, 2rem);
}

/* Texto de cuerpo */
p {
  font-size: clamp(0.9rem, 1vw + 0.5rem, 1.15rem);
}

/* Cómo funciona clamp():
   1. Si "preferido" < "mínimo" → usa mínimo
   2. Si "preferido" > "máximo" → usa máximo
   3. Si está entre ambos → usa preferido (fluido)
   
   El truco: usar vw en el valor preferido para que
   escale con el ancho de la ventana.
   
   4vw + 0.5rem = escala con viewport + base fija
*/

4. Imágenes y media responsivos

La regla de oro: max-width: 100% + height: auto. La imagen nunca será más ancha que su contenedor.

📷

Imagen fluida

max-width: 100%; height: auto;

Se encoge con el contenedor, nunca se desborda

🖼️

object-fit: cover

Rellena el espacio, recorta si es necesario

Mantiene proporción, recorta los bordes

🎬

aspect-ratio: 16/9

Mantiene proporción 16:9 siempre

Proporción fija sin importar el ancho

/* Imagen fluida básica */
img {
  max-width: 100%;  /* Nunca más ancha que su contenedor */
  height: auto;     /* Mantiene la proporción */
  display: block;
}

/* object-fit: controla cómo la imagen llena su caja */
.cover {
  width: 100%;
  height: 200px;           /* Altura fija */
  object-fit: cover;       /* Rellena, recorta si hace falta */
  object-position: center; /* Centra el recorte */
}

/* aspect-ratio: mantiene proporción sin trucos */
.video-container {
  width: 100%;
  aspect-ratio: 16 / 9;  /* Siempre 16:9 */
}

/* <picture> para servir distintas imágenes según pantalla */
<picture>
  <source media="(min-width: 768px)" srcset="grande.jpg">
  <source media="(min-width: 480px)" srcset="mediana.jpg">
  <img src="pequeña.jpg" alt="Descripción">
</picture>

/* ===== MOBILE FIRST: base = 1 columna ===== */
.layout {
  display: grid;
  grid-template-columns: 1fr;
  gap: 10px;
}

/* ===== TABLET: 576px+ → 2 columnas ===== */
@media (min-width: 576px) {
  .layout {
    grid-template-columns: 200px 1fr;
    grid-template-areas:
      "header  header"
      "sidebar content"
      "sidebar aside"
      "footer  footer";
  }
}

/* ===== ESCRITORIO: 1024px+ → 3 columnas ===== */
@media (min-width: 1024px) {
  .layout {
    grid-template-columns: 200px 1fr 180px;
    grid-template-areas:
      "header  header  header"
      "sidebar content aside"
      "footer  footer  footer";
  }
}

📋 Cheatsheet de Responsive Design

Meta viewport

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Obligatorio. Sin esto, el móvil no respeta tus media queries.

Mobile First

@media (min-width: 768px) { }

CSS base = móvil. Media queries agregan estilos para pantallas más grandes.

Unidades fluidas

width: 90%; font-size: 1.2rem;

Usar %, rem, vw en vez de px fijos para que todo escale.

clamp()

font-size: clamp(1rem, 2vw + 0.5rem, 2rem);

Tipografía fluida sin media queries. Mínimo, preferido, máximo.

Imágenes fluidas

img { max-width: 100%; height: auto; }

La imagen nunca desborda su contenedor.

Breakpoints comunes

576px | 768px | 1024px | 1200px

No son reglas fijas. Adaptá los breakpoints a tu diseño.
